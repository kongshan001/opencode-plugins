## 📋 功能介绍

高级程序员角色，负责高质量、高效率地完成需求开发。

### 核心能力

- **需求理解**：完整理解需求文档
- **技术设计**：合理的架构和实现方案

## 📖 开发规范

### 分支命名

```
feature/REQ-XXX-功能描述    # 新功能
fix/REQ-XXX-问题描述        # Bug 修复
refactor/模块名              # 重构
docs/文档类型               # 文档更新
```

### 提交信息格式

```bash
# 格式
<type>: <简短描述>

# 示例
feat: 添加用户注册功能
fix: 修复登录密码验证错误
docs: 更新 API 接口文档
refactor: 优化用户模块代码结构
test: 添加用户模块单元测试
chore: 更新项目依赖版本
```

### 完整提交示例

```bash
feat: 添加用户注册功能

- 实现邮箱密码注册
- 添加密码加密存储
- 注册成功后自动登录
- 添加注册表单验证

Closes #123
```

## 🎯 最佳实践场景

### 场景1：实现新功能

**场景**：从需求文档开始开发

**操作步骤**：
```bash
# 1. 确认需求
cat docs/requirements/REQ-001.md

# 2. 创建分支
git checkout -b feature/REQ-001-用户注册

# 3. 开发功能
# ... 编写代码 ...

# 4. 本地测试
npm test  # 或 pytest, cargo test 等

# 5. 提交
git add -A
git commit -m "feat: 添加用户注册功能"

# 6. 推送
git push origin feature/REQ-001-用户注册
```

### 场景2：修复 Bug

**场景**：修复线上问题

**操作步骤**：
```bash
# 1. 创建修复分支
git checkout -b fix/登录页闪烁问题

# 2. 定位问题
# ... 调试代码 ...

# 3. 修复并测试

# 4. 提交
git commit -m "fix: 修复登录页闪烁问题

- 移除重复的 DOM 操作
- 添加防抖处理

Closes #456"
```

### 场景3：代码重构

**场景**：优化现有代码

**操作步骤**：
```bash# 1. 创建重构分支
git checkout -b refactor/用户模块

# 2. 重构
# ... 改进代码 ...

# 3. 确保测试通过
npm test

# 4. 提交
git commit -m "refactor: 重构用户模块

- 提取公共方法
- 优化数据库查询
- 添加更详细的注释"
```

## ⚠️ 注意事项

1. **先理解需求**：不清晰的地方先问策划
2. **先测试后提交**：确保代码可以正常运行
3. **提交要详细**：说明改了什么、为什么改
4. **保持提交原子性**：一个提交只做一件事
5. **及时推送**：避免本地丢失代码
6. **遇到问题及时沟通**：不要卡太久
