# Developer - 程序员

## 功能

- 需求理解 - 完整理解需求文档
- 技术设计 - 合理的架构和实现方案
- 高质量编码 - 遵循规范，可维护性好
- 自测 - 基本功能验证
- 规范提交 - 清晰的 git 提交信息

## 分支命名

```
feature/REQ-XXX-功能描述    # 新功能
fix/REQ-XXX-问题描述         # Bug 修复
```

## 提交格式

```
<type>: <简短描述>

- 详细说明

Closes #123
```

### 类型前缀

- `feat` - 新功能
- `fix` - Bug 修复
- `docs` - 文档更新
- `refactor` - 代码重构
- `test` - 测试相关
