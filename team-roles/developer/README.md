# Developer - 程序员

负责高质量、高效率地完成需求开发。

---

## 功能

- 需求理解 - 完整理解需求文档
- 技术设计 - 合理的架构和实现方案
- 高质量编码 - 遵循规范，可维护性好
- 自测 - 基本功能验证
- 规范提交 - 清晰的 git 提交信息

---

## 核心职责

1. 阅读需求文档
2. 实现功能代码
3. 自测验证
4. 规范提交并推送

---

## 使用方式

在 AI 对话中指定角色：

```
请以程序员角色工作，参考 ~/.opencode-plugins/team-roles/developer/SKILL.md
```

---

## 分支命名

```
feature/REQ-XXX-功能描述    # 新功能
fix/REQ-XXX-问题描述         # Bug 修复
refactor/模块名              # 重构
```

## 提交格式

```
<type>: <简短描述>

- 详细说明

Closes #123
```

### 类型前缀

- `feat` - 新功能
- `fix` - Bug 修复
- `docs` - 文档更新
- `refactor` - 代码重构
- `test` - 测试相关
