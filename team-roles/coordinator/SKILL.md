# 团队协调者 Team Coordinator

## 📋 功能介绍

团队协调者，负责协调整个团队的工作流程，确保各角色有序协作。

### 核心能力

- **流程管理**：确保开发流程完整执行
- **进度追踪**：了解各角色工作状态
- **问题协调**：解决角色间的冲突
- **质量把控**：确保交付质量

## 🎯 最佳实践场景

### 场景1：完整功能开发

**场景**：从需求到上线

**操作步骤**：

```bash
# Step 1: 策划 - 创建需求
# 参考 planner/SKILL.md
# → 编写 docs/requirements/REQ-001.md

# Step 2: 程序 - 开发功能
# 参考 developer/SKILL.md
# → 创建分支 feature/REQ-001
# → 编写代码
# → 提交

# Step 3: 审核 - Code Review
# 参考 reviewer/SKILL.md
# → gh pr create
# → 等待 review 通过

# Step 4: QA - 编写测试
# 参考 qa/SKILL.md
# → 编写 tests/
# → 确保覆盖率达标

# Step 5: 文档 - 更新文档
# 参考 doc-writer/SKILL.md
# → 更新 docs/api/
# → 更新 docs/core-classes/

# Step 6: 合并推送
git checkout main
git merge feature/REQ-001
git push origin main
```

### 场景2：快速修复

**场景**：紧急 Bug 修复

**操作步骤**：

```bash
# 1. 评估优先级
# 如果是 P0（紧急），可以简化流程

# 2. 开发修复
git checkout -b fix/bug-xxx
# → 修复 bug
# → 提交

# 3. 快速 Review
# → 核心成员 approve 即可

# 4. 测试
# → 运行现有测试

# 5. 合并
git checkout main
git merge fix/bug-xxx
git push
```

### 场景3：大型重构

**场景**：架构调整

**操作步骤**：

```bash
# 1. 策划阶段
# → 编写重构需求文档
# → 评估影响范围

# 2. 设计与评审
# → 技术方案设计
# → 团队评审

# 3. 分阶段实施
# → 拆分为多个小任务
# → 每阶段独立可运行

# 4. 持续 Review
# → 每个阶段都 review

# 5. 完整测试
# → 确保覆盖率不下降
# → 性能测试

# 6. 文档更新
# → 架构文档
# → 核心类文档
```

## 📊 质量标准

| 阶段 | 标准 |
|------|------|
| 需求 | 每个 REQ 有验收标准 |
| 开发 | 代码可运行、无明显问题 |
| Review | 至少 1 人 approve |
| 测试 | 覆盖率达标 |
| 文档 | 与代码同步 |
| 提交 | 推送到远程仓库 |

## ⚠️ 注意事项

1. **不要跳过步骤**：每个流程都要执行
2. **遇到问题及时沟通**：不要卡住
3. **保持可追溯**：每个变更有记录
4. **确保完整交付**：代码+测试+文档
5. **及时推送**：避免本地丢失
